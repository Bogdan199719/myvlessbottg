# Bot Flow

## Первый вход

Пользователь попадает в `/start`, регистрируется в БД и проходит onboarding:

- принятие условий;
- проверка подписки на канал, если включён `force_subscription`;
- переход в главное меню.

## Главное меню

Фактические кнопки:

- `Купить VPN подписку`
- `Купить Telegram Proxy` при наличии активных MTG-хостов
- `Попробовать бесплатно`, если trial включён, ещё не использован и у пользователя не было платной VPN-подписки
- `Мой профиль`
- `Мои подписки`
- `Реферальная программа`, если включена
- `Инструкция`
- `Поддержка`
- `О проекте`
- `Рассылка` только для администратора

## Покупка VPN

Текущий путь:

1. Нажатие `Купить VPN подписку`.
2. Сразу показ списка глобальных VPN-тарифов без отдельного выбора сервера.
3. Выбор способа оплаты.
4. Для `YooKassa` бот запрашивает email для чека, если включена настройка `email_prompt_enabled`.
5. После успешной оплаты бот выдаёт одну постоянную subscription-link `/sub/...`, которая содержит доступы ко всем активным XUI-хостам.

Одиночные XUI-ключи пользователю больше не продаются и не показываются как отдельный товар. Продление старого XUI-доступа из интерфейса также ведёт на покупку глобальной VPN-подписки.

## Пробный период

- Кнопка trial показывается только если `trial_enabled=true`, пользователь ещё не использовал пробный период и у него не было платной VPN-подписки.
- Trial для VPN работает в логике глобальной подписки, а не как отдельный одноразовый ключ.
- При выдаче trial создаются доступы на все активные XUI-хосты, после чего пользователь получает одну `/sub/...` ссылку.
- После окончания trial пользователь может продлить обычную глобальную подписку и продолжить использовать ту же subscription-link без повторного добавления вручную.

## Глобальная VPN-подписка

- Ссылка подписки строится из постоянного `users.subscription_token`: `https://<domain>/sub/<token>`.
- Истечение доступа не удаляет и не меняет этот token. Старая ссылка остаётся ссылкой того же пользователя.
- При продлении глобального тарифа бот переиспользует существующие XUI-клиенты по хостам и обновляет срок действия; пользователю достаточно обновить подписку в клиентском приложении.
- Если в клиентском приложении уже добавлена `/sub/...` ссылка, добавлять новую ссылку после продления не нужно.
- При продлении бот нормализует состояние клиентов в 3x-ui и сбрасывает устаревший флаг исчерпания трафика, чтобы включённая после оплаты учётная запись не оставалась помеченной как expired.

## Покупка Telegram Proxy

1. Пользователь открывает `Купить Telegram Proxy`.
2. Выбирает MTG-хост.
3. Выбирает тариф.
4. Оплачивает.
5. Получает `tg://proxy` ссылку и кнопку подключения.

## Способы оплаты

- `YooKassa` — через независимую интеграцию YooKassa API: redirect на внешнюю оплату, webhook `/yookassa-webhook`, повторная проверка платежа через API перед выдачей доступа.
- `Telegram Stars` — через встроенные платежи Telegram.
- `CryptoBot` — invoice + webhook `/cryptobot-webhook/<token>`.
- `P2P` — пользователь оплачивает вручную, нажимает подтверждение и получает экран ожидания с кнопками перехода в меню и в поддержку; администратор подтверждает или отклоняет заявку.

Для YooKassa нужен email покупателя для фискального чека: либо бот запрашивает его перед оплатой, либо в настройках указан реальный fallback `receipt_email`. Placeholder-адреса не считаются валидной настройкой.

## Разделы после покупки

### Мой профиль

Показывает:

- username;
- сумму трат;
- статус VPN;
- ссылку VPN-подписки;
- переход к списку подписок и proxy.

### Мои подписки

Содержит:

- `Моя VPN подписка`, если у пользователя есть глобальный тариф или активный global trial;
- ссылку глобальной VPN-подписки для копирования и подключения;
- список Telegram Proxy;
- trial больше не выводится как отдельный блок, если он выдан как глобальная подписка.

Кнопка подробного списка XUI-ключей в пользовательском боте убрана: пользователь покупает и продлевает именно VPN-подписку, а не отдельные серверные ключи.

## Инструкции

Основной клиент для всех платформ, кроме Linux:

- Happ для `Android`
- Happ для `iOS`
- Happ для `Windows`
- Happ для `macOS`

Linux описан отдельно и не приводился к Happ.

## Поддержка

Если настроен support-бот и support group:

- пользователь пишет support-боту;
- для него создаётся forum topic;
- переписка между личкой и группой пересылается в обе стороны;
- тикет и сообщения сохраняются в SQLite;
- если topic удалён или Telegram отправляет сообщение не в тот thread, бот помечает тикет на восстановление и создаёт новый topic на следующем сообщении пользователя;
- краткая история последних сообщений публикуется в восстановленный topic, а полная история остаётся доступной в админке.
