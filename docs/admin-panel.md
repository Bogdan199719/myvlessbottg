# Admin Panel

## Доступ

Админка поднимается тем же процессом, что и бот, и по умолчанию слушает порт `1488`. Авторизация выполняется по логину и паролю из `bot_settings`. Пароль хранится в хеше; legacy plaintext при успешном входе автоматически обновляется до hash.

Защита, которая реально есть в коде:

- session cookie `HttpOnly`;
- `SameSite=Lax`;
- таймаут сессии `8 часов`;
- CSRF для всех `POST`, кроме webhook-роутов;
- ограничение попыток входа по IP.

## Разделы

### Dashboard

- сводная статистика;
- проблемные пользователи;
- последние транзакции;
- графики по дням.

### Users

- список пользователей и их ключей;
- ban / unban;
- revoke всех ключей;
- полное удаление пользователя;
- ручная выдача подписки;
- диагностика по конкретному пользователю;
- экспорт CSV.

### Keys

- сгруппированный список ключей по пользователям;
- корректировка срока;
- синхронизация connection string с 3x-ui;
- исправление параметров клиентов;
- комплексное обслуживание;
- показ и копирование VPN-ключа, proxy-link или subscription-link.

### Settings

- общие тексты и ссылки;
- токены ботов и параметры панели;
- payment settings;
- trial и referral settings;
- backup/import с отдельными опциями включения и применения `.env`;
- XUI-хосты и MTG-хосты;
- планы;
- payment method rules;
- включение/отключение global plans.

Support-бот при этом продолжает работать отдельно от админки:

- создаёт `forum topic` в группе поддержки;
- сохраняет тикеты и историю сообщений в SQLite;
- восстанавливает тикет при потере Telegram topic.

Отдельно важно:

- настройка `Спрашивать Email перед оплатой YooKassa` относится только к оплате через YooKassa;
- если опция выключена, бот не спрашивает email перед оплатой и использует fallback `receipt_email` для чека;
- fallback `receipt_email` должен быть реальным email, а не placeholder вроде `example@example.com`; иначе YooKassa скрывается из доступных способов оплаты;
- VPN-тарифы для пользователей работают как глобальная подписка `ALL`: пользователь получает одну `/sub/...` ссылку, а подробный список отдельных XUI-ключей в пользовательском боте не показывается.

### Updates

- проверка версии по GitHub;
- force-update приложения.

## Админские действия с прод-эффектом

Особенно аккуратно использовать:

- удаление хоста;
- удаление пользователя;
- revoke ключей;
- backup import;
- update из панели.

Эти действия не только меняют БД, но и могут удалять клиентов на внешних панелях или перезапускать приложение.
